Lecture: The OSI Model

Introduction: Solving Network Chaos

▼

In the early days, network development was chaotic. Each vendor created their own proprietary systems that couldn't communicate with others. To solve this, the International Organization for Standardization (ISO) developed the Open Systems Interconnection (OSI) model, released in 1984. The OSI model provides a standardized, layered framework that allows hardware and software from different vendors to work together seamlessly. It's also an excellent tool for teaching and understanding how networks operate.

The 7 Layers of the OSI Model

▼

The OSI model consists of seven distinct layers, each with a specific function. Data must pass through all layers in order, from top to bottom on the sending device and from bottom to top on the receiving device. A popular mnemonic to remember the layers is: "All People Seem To Need Data Processing".

  • Layer 7: Application: The layer closest to the user. Provides network services directly to user applications like web browsers (HTTP), email clients (SMTP), and file transfer (FTP).
  • Layer 6: Presentation: Formats and translates data to be presentable for the Application layer. It handles data encryption, compression, and character encoding (e.g., ASCII, EBCDIC).
  • Layer 5: Session: Establishes, manages, and terminates sessions between two hosts. It ensures that the communication link between applications is maintained.
  • Layer 4: Transport: Provides reliable or unreliable data transfer between end systems. It segments data into smaller chunks and uses protocols like TCP (reliable, connection-oriented) and UDP (unreliable, connectionless).
  • Layer 3: Network: Handles logical addressing (IP addresses), path determination, and routing of data packets across different networks. IPv4 and IPv6 operate at this layer.
  • Layer 2: Data Link: Formats data into frames and provides reliable transit across a physical medium. It handles physical addressing (MAC addresses), error detection, and flow control. Ethernet is a key Layer 2 protocol.
  • Layer 1: Physical: Defines the physical characteristics of the network, such as cables, connectors, voltage levels, and data rates. It's the layer you can physically touch.

Encapsulation, De-encapsulation, and PDUs

▼

The process of data moving down the OSI model layers is called encapsulation. Each layer adds its own header (and sometimes a trailer) to the data it receives from the layer above. The reverse process on the receiving end is de-encapsulation.

The Encapsulation & De-encapsulation Process

The "packet of information" at each layer is called a Protocol Data Unit (PDU). These have different names at different layers:

  • Layers 7, 6, 5: Data
  • Layer 4 (Transport): Segments (for TCP) or Datagrams (for UDP)
  • Layer 3 (Network): Packets (e.g., IP Packets)
  • Layer 2 (Data Link): Frames (e.g., Ethernet Frames)
  • Layer 1 (Physical): Bits

OSI Model in Action with Wireshark & tcpdump

▼

We can observe the OSI model's layers using network analysis tools. Wireshark provides a graphical interface, while `tcpdump` is a powerful command-line utility. Capturing an HTTP request reveals the layered structure:

  1. The Ethernet Frame (Layer 2): Contains source/destination MAC addresses.
  2. The IP Packet (Layer 3): Contains source/destination IP addresses.
  3. The TCP Segment (Layer 4): Contains source/destination ports.
  4. The HTTP Data (Layer 7): Contains the browser's request.

Here is a `tcpdump` command to capture similar traffic. Click "Run Code" to see a simulation of the output.

$ tcpdump -i any 'tcp port 80 and host cisco.com'

Explanations

             

                

                    

                        
Explanation for Question 1

                        ▼
                    

                    

                        
The correct matches are: Network Layer handles IP addressing and routing. Application Layer handles protocols like HTTP. Physical Layer deals with physical media like cables. Transport Layer provides reliability with TCP. Data Link Layer manages MAC addresses.

                    

                

                 

                    

                        
Explanation for Question 2

                        ▼
                    

                    

                        
The PDUs are named: Layer 4: Segment (for TCP). Layer 3: Packet. Layer 2: Frame. It's important to use the correct terminology (e.g., "IP packet," not "IP frame").

                    

                

                

                    

                        
Explanation for Question 3

                        ▼
                    

                    

                         
Encapsulation is the process on the sending device where each layer adds its own header to the data from the layer above it, wrapping it for transmission. De-encapsulation is the reverse process on the receiving device, where each layer strips off its corresponding header to pass the data up to the next layer.

                    

                

                

                    

                        
Explanation for Question 4

                        ▼
                    

                    

                        
A good answer includes: Layer 7: The browser creates an HTTP GET request. Layer 3: An IP packet is created with the source IP (your computer) and destination IP (Google's server). Layer 2: An Ethernet frame is created with the source MAC address (your network card) and the destination MAC address (your router's MAC address).
